## Run against a host daemon

> **Note:** The commands below assume you are already inside the scaffold project
> directory (the directory that contains `bots/` and `pacto-bot-api.toml`).
> If you are running from a parent directory, use `cd {{project_dir_name}}` first.

1. Make sure `pacto-bot-api` is running on the host. The default Unix socket is
   at `~/.local/share/pacto-bot-api/pacto-bot-api.sock`.
2. Change into the bot directory:
   ```bash
   cd bots/{{bot_id}}
   ```
3. Install the bot package. The `pacto-bot-sdk` dependency will be pulled from
   PyPI automatically:
   ```bash
   pip install .
   ```
4. Run the handler:
   ```bash
   python {{bot_id_snake}}.py
   ```

## Install as a systemd service

The generated `bots/{{bot_id}}/systemd.service` file runs the bot as a
non-root `pacto-bot` user against a host daemon socket.

1. Create the runtime user and group (skip if you already have one):
   ```bash
   sudo groupadd --system pacto-bot
   sudo useradd --system --no-create-home --gid pacto-bot pacto-bot
   ```

2. Copy the bot to `/opt/{{bot_id}}` and install its dependencies:
   ```bash
   sudo cp -r bots/{{bot_id}} /opt/{{bot_id}}
   sudo pip install /opt/{{bot_id}}
   sudo chown -R pacto-bot:pacto-bot /opt/{{bot_id}}
   ```

3. Install the systemd unit:
   ```bash
   sudo cp /opt/{{bot_id}}/systemd.service /etc/systemd/system/{{bot_id}}.service
   sudo systemctl daemon-reload
   sudo systemctl enable --now {{bot_id}}
   ```

4. Check status and logs:
   ```bash
   sudo systemctl status {{bot_id}}
   sudo journalctl -u {{bot_id}} -f
   ```

## Run the default compose stack

From the project root:

```bash
docker compose up --build
```

This starts the bot and the `pacto-bot-api` daemon. By default the daemon
connects to a relay at `ws://localhost:7000`; use the `relay` profile (or set
`PACTO_RELAY_URL`) to point to an internal or external relay.

Use the `with-bunker` profile to add the NIP-46 bunker, Postgres, and Redis:

```bash
docker compose --profile with-bunker up --build
```

The daemon and bot share a Docker volume for the Unix socket, so there is no
host socket to create or permission-manage.

### Images required for the stack

The daemon image is pulled from `ghcr.io/logicminds/pacto-bot-api:latest`. The
relay and bunker images are pulled from the `pacto-dev-env` GHCR repository:
`ghcr.io/covenant-gov/pacto-dev-env/nostr-relay:main` and
`ghcr.io/covenant-gov/pacto-dev-env/nip46-bunker:main`.

These images are built automatically by the `Release container images` workflow
in `covenant-gov/pacto-dev-env`. If the packages are private, ask a repository
admin to make them public in GitHub Container Registry settings before pulling.

## Use an internal Nostr relay

```bash
PACTO_RELAY_URL=ws://nostr-relay:8080 docker compose --profile relay up --build
```

The internal relay is also exposed on the host at `ws://localhost:7000`.

## Use an external Nostr relay

Set `PACTO_RELAY_URL` to your relay URL and start the default stack (no relay
profile):

```bash
PACTO_RELAY_URL=wss://your-relay.example.com docker compose up --build
```

## Security notes

- The `nsec` signing backend is intended for **development only**. Production
  deployments should use `bunker_local` or `bunker_remote`.
- The generated `pacto-bot-api.toml` contains signing material and should never
  be committed to version control. It is ignored by `.gitignore` and
  `.dockerignore` by default.
- Never commit real signing material, `nsec` values, or daemon secrets to
  version control.
